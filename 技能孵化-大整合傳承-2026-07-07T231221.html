工作階段 S1|GEPA 首輪:從零到訓練集 98.15、保留集 91.13(13 段)

【S1-001|開場需求】 我們在 data 底下有 input 跟 output。input 裡面每一個資料夾都是一批案件,可能是 PDF、可能是 DOCX,是每一個人的起訴、不起訴跟偵查結果的書類;裡面會有個資、犯罪行為等等。 一開始系統是沒有「人口販運案件、未遂犯、幫助犯、重大刑事案件、國土保育案件」這些資料夾的,連資料夾名稱都不知道——是人類專家判斷之後,才幫它分好這些資料夾。 分好資料夾之後,還要對應 output 資料夾裡的 Excel 表:它有五個頁簽,對應剛剛人工建立的五個分類。每一筆個案都要有上面的欄位——例如它是臺北地檢還是桃園地檢、年份是 114 年、字別是偵字還是偵緝字(這些都有機關自己的代號)、被告姓名(去識別化)——最重要的是要判它屬於起訴、不起訴還是緩起訴。 每個頁簽下面要分類的項目又不一樣:人口販運是「多選題」,有性剝削、勞力剝削、器官摘取、人流處置;國土保育通常是「單選題」,可能是野生動物保育法或森林法;幫助犯也有一些類別。 這些 know-how 放在「客戶提供的原始資料-統計書類」裡,有兩個檔案:一個 TXT 是領域專家口述的前情提要;另一個是人類專家訪談事務官整理出來的 HTML,裡面寫了 AI 要怎麼判讀起訴書/不起訴書、怎麼判未遂、怎麼判幫助犯(例如刑法第 30 條)。重大刑案有指定法條清單;國土保育也有指定條號,例如森林法第 51、53、54 條——52 條就不是。每一個欄位都是專家指定的法條專業領域,這些都很重要。 現在請你上網查 pi.dev 怎麼製作技能。你的電腦裡已經有 pi,用「pi 空格 -p」可以驅動地端 31B 模型。查清楚之後,在 .pi/ 底下可以做 skills。 針對輸入輸出的需求,你要整理出乾淨的資料夾:訓練資料要切割成「訓練集」跟「保留集」,放不同資料夾。Fable(你)有能力做出標準正確答案;但技能訓練的目標對象是 pi -p 的 31B 小模型。我希望你用 Fable 把資料夾轉成 AI 友善的純文字——PDF 轉純文字、Excel 也轉 AI 友善格式。一旦轉好,你就可以自由切割保留集和訓練集來訓練技能。 訓練技能時,請上網查一個叫 GEPA 的 GitHub。GEPA 可以多輪疊代:評審請用 Fable subagent、乾淨上下文來打分;然後照 GEPA 的精神,每一輪針對 pi -p 這個「選手」疊代。GEPA 就像教練,不斷教選手進步。技能越進步,它對輸入輸出的考題就越厲害。 所以你要清楚告訴我:考題有幾題、訓練集訓了幾題、訓練完保留題過了幾題、幾分、情況怎麼樣。評審、裁判用 Fable subagent 做,選手是 pi -p。每一輪疊代都要清清楚楚講:幾分、做了什麼事;疊代過程的技能或腳本都要留下來。技能如果純文字 SKILL.md 就能跑,就這樣跑;要加腳本的話,pi.dev 可以加技能腳本,請用 bun 寫 TypeScript 放進去。 技能疊代完成之後,它就有能力把 AI 友善的純文字輸入,做成 AI 友善的純文字輸出——輸出就是剛剛講的試算表,每個欄位都要有,等於是把一個案件做標籤分類,輸出成不同頁簽的試算表。最後你再用腳本工具把它變成真正的 Excel 格式。工具一律優先用 bun + TypeScript 製作。每一輪都報分數和原因。
🎯 一段 prompt 就把整個孵化案定調:輸入輸出樣態、專家 know-how 在哪、選手(31B)/教練(GEPA)/評審(Fable subagent)三角、訓練/保留集切割、工具鏈(bun+TS)、驗收語言(每輪報分數)。💡 之後三輪所有工作都是在執行這一段的骨架——開場需求寫得越完整,後面返工越少。⚠️ 地雷:「Fable 轉 AI 友善純文字」是純格式轉換(轉檔),不是叫 Fable 讀懂內容;對接的人只要說「把這個資料夾轉純文字」就夠。

【S1-004|Dynamic Workflow 重構指示】 有幾個問題要講。你的技能出來之後,很有可能要搭配腳本做多步驟產出——當然包含 OCR、Word、PDF 轉 AI 友善純文字,這些你都要做,而且要包含在技能裡面;不能搞半天這些不在技能裡,那就穿幫了。所以技能一定要有「AI 友善轉換」的能力,轉換完最後還要變成 Excel。 問題來了:31B 在做這些的時候,請你上網查 Claude Code 的 Dynamic Workflow。它的原理是用 JS 腳本控制整個工作流——技能在腳本下面被控制。多步驟、多腳本,就有機會用 JS 控制工作流,不會讓 Agentic AI 亂掉、有步驟沒做到、做得不夠好、或沒有平行處理。這些都可以用 Dynamic Workflow 的原理解決。 我們希望你把這個概念用在本案:31B 的技能啟動之後,有一個 bun TypeScript 輔助腳本實現 Dynamic Workflow。腳本裡做多步驟拆解:從原始資料開始,要怎麼呼叫大模型——可以用 pi 空格 -p 的方式呼叫,因為它有 Agentic AI 的能力——而且最好平行做,不然會很慢。透過這些方式,就有機會把大量資料處理好。 另外一個困惑:如果資料不是只有幾個檔案,而是十幾個、100 個、300 個檔案,你的壓力測試過得了過不了?要跑多久?所以做完之後,幫我測:多快(速度數字)、多省資源、轉換成最終檔案多精準、品質多好——給我一份測量報告。每一輪疊代時,要根據這些 baseline 跟測量報告的差異,才能疊代下一代。 我猜最後實際使用也不會一個檔案一個檔案分類,那樣沒意義;應該是批次做,一次 100 或 300 個檔案。例如一個地檢署多個檔案全部放進某個資料夾,技能一啟動,就跑 Dynamic Workflow 裡寫好的腳本完成這件事。總之怕你工作流設計錯誤——上網查清楚之後,照 Claude Code Dynamic Workflow 的精神,好好壓力測試、重構成能夠很穩健地用 31B、用 pi -p 完成的東西。
🎯 本案最重要的架構指令:①技能必須自包含(含轉檔,否則「穿幫」);②腳本控流程、模型只判讀;③要壓測、要 baseline。💡 直接催生了 pipeline.ts 四階段管線與 +4.90 的最大單輪躍升。⚠️ 地雷:小模型撐不起自主編排——步驟會漏、不會平行、每次路徑不同;把控制流交給腳本才可重現。

【S2-002|抓取五類案件相關法規法條(核心需求)】 附件:`data/2026-06-30-客戶提供的原始資料-統計書類/17-統計處書類裁判書標籤判讀-需求情境拷貝.html` 想要抓全國法規資料庫裡相關的法條,存在適當的資料夾底下。 背景思考(自問自答釐清目的):要怎麼抓?抓哪些範圍?我猜是刑法相關的。為什麼要抓?目的是做「對照表」,用來做分類。為什麼抓了法條就可以分類?因為像國土保育這類案件,起訴書裡會寫到它涉犯某一條法;看到這條法,就可以把案件分類到國土保育或某一類案件。輸入會是一份 PDF 或 Word,輸出就是根據內文的涉犯法條,對照到全國法規資料庫裡的條文,完成分類——是這個概念。 訪談時法務部的領域專家說過:「法條對照表是核心,所有判讀都是從法條出發。」但我們並不具備法律專業,不知道對照表要怎麼對照。請 Fable 幫我們上網查清楚——你有 Claude in Chrome 可以用,也可以用 Bun/TS 寫程式碼大量抓取,都可以。 抓取範圍(五個類別):未遂犯、幫助犯、重大刑案、人口販運類別、國土保育案件。 要求: - 按照全國法規資料庫的「法規 → 法條」階層形式,把這五類相關的法規與法條抓下來存檔,放在本案目錄底下。 - 資料夾要標「臺灣時間的時間戳」標籤。 - 檔名要有系統性,能與全國法規資料庫對照。 - 要做一個「盤點表」,不能漏抓:用 metadata JSONL 或相關描述講清楚——哪些已抓、哪些還沒抓、幾月幾號抓的,都要記錄清楚,可供對照核對。 判讀邏輯舉例:如果個案的涉犯法條包含「刑法第 30 條」(幫助犯條文),幫助犯就成立——這就是要抓法條的原因。不只抓法條本文,還要做「條目清單」:例如涉及「重大刑案 19 罪名清單」的就歸重大刑案;毒品危害防制條例第 4 條第 1、2 項與第 5 條第 1 項,要判斷毒品數量是否達 1,000 公克以上,達標才算重大刑案。 對照表 DSL:附上的 HTML 裡有我們專家訪談整理的原理原則。請透過這些原理原則,做出一個對照表的 DSL(領域專屬語言),用 JSON 或 JSONL 表示。一旦有了這個完整的對照表,AI 回答時照表比對,就幾乎不會錯。
🎯 把專家一句「法條對照表是核心」變成資料資產:官方 API 快照(18 部法規、95 條文)+盤點表(inventory.jsonl,防漏抓)+對照 DSL(mapping-dsl.json,含語意護欄)。💡 這段 prompt 的「自問自答釐清目的」示範了怎麼把模糊需求想清楚再下指令。⚠️ 地雷:抓資料一定要盤點表+時間戳,否則三個月後沒人知道抓了什麼、缺了什麼、是哪天的版本。

// mask.ts — 姓名去識別化(20 行救回 37 個錯題)
export function maskName(name: string): string {
  const s = String(name ?? "").trim().replace(/\s+/g, " ");
  if (s.includes("○")) return s.replace(/\s/g, "");      // 已匿名:照抄
  if (/^[A-Za-zÀ-ÿ'. -]+$/.test(s)) return s.split(" ")[0] + "○○"; // 拉丁姓名
  const chars = [...s.replace(/[\s·]/g, "")];
  if (chars.length === 2) return chars[0] + "○";
  if (chars.length === 3) return chars[0] + "○○";
  return chars[0] + chars[1] + "○".repeat(chars.length - 2);  // 4 字雙姓
}

// orgn.ts — 22 家地檢對照表(12 家答案卷實證+10 家區域規律推導)

  雲林:{code:"2223",verified:true}, /* …221x 北、222x 中、223x 南、224x 東離島… */ };
export function resolveOrgn(docText: string) {
  const head = docText.slice(0, 600).replace(/\s+/g, "").replace(/台/g, "臺");
  const variants = [head, head.replace(/(.)\1+/g, "$1")];   // 折疊 OCR 疊字
  // match /[臺福][灣建](○○)地方檢察署/ → 查表 → 覆寫;查不到 → 留空+警告,不整案失敗
}

006|為什麼能突破(三輪共同的答案)
